## 6. 重要な設計メモ（明日忘れないよう）

- **聖女ジャンヌ v2 の検出方法**: `state.defenseCard?.name === '聖女ジャンヌ' && state.attackRevealed.length === 0` で passive 判定。新 state 不要。
- **`lastRoundBonuses`**: Phase 1 + Phase 2 items を合算した単一 RoundBonuses として engine 側で生成。UI は `TrophyBonusBreakdown` が `bonuses.items[]` を順次表示。`winnerSide` は `RoundBonuses` 自体に含まれる（`advanceToNextRound` で `state.roundWinner` が null リセットされるため）。
- **`roundStartSnapshot`**: 劣勢逆転判定用にラウンド開始時のファン値のみ記録（deck/bench は「勝利瞬間」の現在値で判定）。
- **`trophyBreakdownDuration(n)`**: `400 + (n+1)*300 + 400` で動的スケール。
- **git 状態事前確認ルール**: destructive op（rebase/force-push）前に `git log --oneline`, `git status`, `git branch -vv`, `git fetch origin` で確認。記憶ファイル `memory/feedback_git_state_precheck.md` 参照。
