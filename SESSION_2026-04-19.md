## 6. 重要な設計メモ

### ALT 発行フロー（新モデル）
1. KC 勝利/敗北 → inline result 画面表示
2. 「ALT X 受け取る」ボタンタップ → `addTotalAlt(fansToAlt(playerFans))` + `triggerEarnEffect(X)` + `altClaimed=true`
3. ボタン「✅ 受け取り済」に変化
4. 「次のステージへ/ホームに戻る」タップ → `handleFinish`
   - ステージ初クリアなら `currentStage.altReward` を別途 `addTotalAlt` 
   - `setLastResult({ score: playerFans, label: 'FAN', altAlreadyGranted: true })`
   - `navigate('/result')`
5. ResultPage: 「N FAN」表示、ALT 加算スキップ、「✨ ALT はバトル画面で受け取り済」表示

### fansToAlt レート切替
将来イベント対応は `getFanToAltRate()` に TODO コメント済み。Supabase or localStorage からイベント情報を取得して `return event.altBonusRate;` で上書き。

### starter デッキの allNames 制約
全 10 starter 全て `allNames = 15` で揃った。今後 starter を編集する際は:
- themeCards + trumpCard + noiseCards の合計を必ず 15 に揃える
- `buildStarterDeck` は `fillNoise: false` なので 15 未満なら console.warn が出る
- 新 starter 作成時はこれを確認

### 聖女ジャンヌ v2 の検出方法
`state.defenseCard?.name === '聖女ジャンヌ' && state.attackRevealed.length === 0` で passive 判定。kk 提案の `subBattleFirstAttackDebuffed` フラグを新設する代わりに既存の `attackRevealed.length === 0` で同等の挙動を得ている。

### git 状態事前確認ルール
destructive op（rebase/force-push）前に `git log --oneline`, `git status`, `git branch -vv`, `git fetch origin` で確認。記憶ファイル `memory/feedback_git_state_precheck.md` 参照。
