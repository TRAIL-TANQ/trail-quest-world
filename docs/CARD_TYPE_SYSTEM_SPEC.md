## 1. 分類体系（6 分類 + 2 サブ）

### 1.1 分類定義

| # | 分類 | サブ | 既定発動ゾーン | 主要な役割 |
|---|---|---|---|---|
| 1 | 人物 | — | ベンチ | キャラクター、passive の核。 |
| 2 | 生物 | 2-1 動物 | ベンチ / スタック（カード毎に柔軟） | 群れ・連鎖の基点。単体でも使えるが複数で強くなる。 |
|   |   | 2-2 植物 | ベンチ | 補助・成長。現状 1 枚（光合成）。 |
| 3 | 武器・兵器 | 3-1 武器 | スタック | 攻撃時に積む。人物の火力源。 |
|   |   | 3-2 兵器 | スタック | 大型破壊兵器。SSR 中心。 |
| 4 | 拠点・建築 | — | ベンチ | 防御 passive・場の制御。 |
| 5 | イベント・事件 | — | 公開時発動 | 歴史的転換点。進化・除外・再構成を担当。 |
| 6 | 思想・技術 | — | 公開時発動 / ベンチ（柔軟） | 最大カテゴリ。シナジーの素材、法典・理論・作品・発明など。 |

### 1.2 発動ゾーン定義

```
┌─────────────────────────────────────────────────────────┐
│ ベンチ（Bench）                                          │
│   常駐ゾーン。passive 効果をラウンド継続で発揮。         │
│   例: 織田信長、安土城、ナポレオン法典、オオカミ          │
├─────────────────────────────────────────────────────────┤
│ スタック（Stack） ← 新概念                                │
│   攻撃フェーズに積み上げる。武器・兵器・一部の動物。      │
│   「攻撃の道具・手段」として単発 or 連鎖的に消費される。 │
│   例: 鉄砲、聖剣、白百合の盾、リトルボーイ、ピラニア      │
├─────────────────────────────────────────────────────────┤
│ 公開時発動（Reveal）                                     │
│   カードを公開した瞬間に効果発動（1 回）。              │
│   イベント・事件・瞬間的な行動・一部の思想カード。       │
│   例: 本能寺の変、焚書坑儒、火刑、トリニティ実験、紙     │
└─────────────────────────────────────────────────────────┘
```

### 1.3 柔軟ゾーン（2-1 動物 / 6 思想・技術）の判定基準

カード毎に以下の基準で判定する：

**ベンチ常駐向き**：
- passive 的に味方を強化する（相対性理論の論文、設計図、十二単）
- 場にいる間、継続的に効果を発揮する（サバンナ、アマゾン川との共生種）

**スタック向き**：
- 「攻撃の瞬間」に効果が意味をもつ（ピラニア、鉄砲、大砲）
- 複数枚重ねる設計意図がある（ナポレオン法典は例外的にベンチ stack）

**公開時発動向き**：
- 1 回限りで強烈な効果（本能寺の変、焚書坑儒、マンハッタン計画）
- 除外・再構成・進化トリガー

## 2. 現行実装との差分

### 2.1 現行カテゴリ enum

`client/src/lib/types.ts`:
```ts
export type CollectionCategory =
  'great_people' | 'creatures' | 'world_heritage' | 'inventions' | 'discovery';
```

`client/src/lib/knowledgeCards.ts`:
```ts
export type CardCategory =
  'great_person' | 'creature' | 'heritage' | 'invention' | 'discovery';
```

### 2.2 新体系との対応マップ

| 現行 (CollectionCategory) | 新体系 | 備考 |
|---|---|---|
| `great_people` | 1. 人物 | ほぼ 1:1 |
| `creatures` | 2-1 動物 / 2-2 植物 | **サブカテゴリ追加が必要**（現状は混在） |
| `world_heritage` | 4. 拠点・建築 | ラベル変更 |
| `inventions` | 3 武器・兵器 / 5 イベント・事件 / 6 思想・技術 | **最大の分割対象**。現状は実質「その他全部」バケット |
| `discovery` | 6. 思想・技術（一部 5 へ） | 「発見」は理論系のみ、イベント混在は分離必要 |

### 2.3 発動ゾーンの現状

- **ベンチ passive**: 実装済み（多数。SYNERGY_MAP / CARD_STAT_OVERRIDES で駆動）
- **スタック**: **概念として未実装**。現状は同名枚数カウントや「stackable_xxx」効果 ID で暗黙的に扱われているのみ
  - `knowledgeCards.ts` L212-213 の `stackable_fission_bonus` / `stackable_defense` は【未実装】コメント付き
  - スタック専用 UI・ゾーン描画・上限制御は存在しない
- **公開時発動**: `reveal_xxx` 系の effect ID で実装されているが、**任意発動の UI 統一が必要**

## 6. kk への質問リスト（意思決定待ち）

以下は Claude Code の調査だけでは判断しきれず、kk の意思決定が必要な項目。

### Q1. 植物サブカテゴリの運用方針
現状「光合成」1 枚のみ。以下のどれを選ぶ？
- (a) 植物サブカテゴリを temporarily 廃止し、光合成を 6 思想・技術へ吸収
- (b) 植物カードを増やす計画がある（糸杉を植物化、竹・桜など追加）
- (c) 現状維持（1 枚のみのサブカテゴリ）

**推奨**: (b) — 自然・生態系デッキの拡張余地として温存

### Q2. 「道具」カードの分類境界
グライダー・風洞・リンゴ・プリズムなどは「武器 or 思想・技術」どちらに分類する？
- 武器（3-1）に入れるなら発動ゾーン＝スタックになり、passive 的な使い方ができなくなる
- 思想・技術（6）に入れるならスタックで使えなくなる

**推奨**: ライト兄弟のグライダー/風洞は **思想・技術**（人物 trigger の bench passive 素材として維持）。キャラベル船は思想・技術。リンゴはニュートン trigger だから思想・技術。

### Q3. 「宝石」系 5 枚（ダイヤ、ルビー、サファイア、エメラルド、アメジスト）の役割
